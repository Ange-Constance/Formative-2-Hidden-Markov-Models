================================================================================
DATA FILE FORMAT REQUIREMENTS
================================================================================

Each CSV file should contain the following columns:
- timestamp (or time)
- accel_x
- accel_y
- accel_z
- gyro_x
- gyro_y
- gyro_z

Metadata.txt should contain:
- Phone Model: [e.g., iPhone 14, Samsung Galaxy S21]
- Sampling Rate: [e.g., 50 Hz, 100 Hz]
- Date Collected: [YYYY-MM-DD]
- Collector Name: [Team member name]

================================================================================
NAMING CONVENTIONS
================================================================================

1. Activity folders: lowercase (standing, walking, jumping, still)
2. CSV files: activity_XXX.csv (where XXX is a 3-digit number)
3. Member folders: member1, member2 (or use actual names if preferred)
4. All Python files: lowercase with underscores (snake_case)
5. All notebooks: numbered prefix for ordering

================================================================================
NOTES FOR TEAM COLLABORATION
================================================================================

1. Each team member should:
   - Create their own folder under data/ with the same internal structure
   - Fill in metadata.txt with their phone and sampling rate information
   - Ensure CSV files follow the same column naming convention
   - Aim for ~25 samples each (distributed across 4 activities)

2. Before pushing to repo:
   - Verify all CSV files are properly formatted
   - Check that metadata.txt is filled out
   - Ensure sampling rates are harmonized (discuss with team)

3. For unseen data:
   - Collect additional samples after initial model training
   - Place in data_unseen/ folder with same structure
   - Used for final model evaluation only
